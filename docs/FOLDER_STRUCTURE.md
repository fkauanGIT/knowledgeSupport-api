## Where to put what (quick reference)

| I'm creating... | It goes in... |
|---|---|
| A new business concept (e.g. `Solution`) | `domain/model/` |
| A new operation the system offers | interface in `application/port/in/` + implementation in `application/service/` |
| A new need for something external (database, API, email) | interface in `application/port/out/` + adapter in `adapter/out/<channel>/` |
| A new REST endpoint | `adapter/in/web/` (controller + records) |
| A new inbound channel (webhook, scheduler, queue) | `adapter/in/<channel>/` |
| The JSON format of an external system | inside that system's adapter (e.g. `adapter/out/jira/`) — never in the domain |
| Conversion between an external format and the domain | a `*Mapper` inside the boundary's adapter |
| Configuration/secret (token, URL) | `.env` (real value) + `.env.example` (template) + `application.yaml` (placeholder) |

## Naming conventions

- **Inbound ports**: `<Verb><Concept>UseCase` — e.g. `CreateStandardUseCase`, `ListCalledsUseCase`.
- **Outbound ports**: `<Concept><Role>Port` — e.g. `StandardRepositoryPort`, `CalledProviderPort`.
- **Adapters**: `<Concept><Channel>Adapter` — e.g. `StandardPersistenceAdapter`, `JiraCalledAdapter`.
- **Mappers**: `<Source/Concept>Mapper` — a `final` class with static `toDomain`/`toEntity` methods.
- **Web DTOs**: `<Concept>Request` / `<Concept>Response` (records).
- **External payloads**: system prefix — `Jira*`, in the future `Chatwoot*`.

## Quick sanity check (before opening a PR)

1. Does `domain/` import anything beyond pure Java and other domain classes? ❌
2. Does `application/` import anything from `adapter/`, JPA, Jackson, or web? ❌ (current only exception: Spring's `@Service` annotation)
3. Does any external format (`Jira*`, `*JpaEntity`) show up outside its own adapter? ❌
4. Any hardcoded secret in code or a committed yaml? ❌ (always `.env`)
5. Does the commit follow Conventional Commits (`feat:`, `fix:`, `chore:`...)? ✅ (see [CONTRIBUTING.md](../CONTRIBUTING.md))
